Issues Found:
High Severity

Recommendation 1:

The code is susceptible to SQL injection attacks due to untreated string concatenation in SQL queries.

Suggested Improvement: Use parameterized queries to prevent SQL injections, such as using placeholders with cursor's `execute` method.
High Severity

Recommendation 2:

The database connection should be closed properly even when an exception occurs to ensure resources are released.

Suggested Improvement: Use a `with` statement for database connections, which ensures that connections are closed automatically.
Medium Severity

Recommendation 3:

Error messages are printed directly, which can expose sensitive information.

Suggested Improvement: Log error messages using a logging framework rather than printing them, allowing better control over log levels and destinations.
Medium Severity

Recommendation 4:

Inefficient handling of file paths and potential errors due to reliance on `__file__` for determining relative data paths.

Suggested Improvement: Consider using environment variables or configuration files for specifying data directories, increasing flexibility and robustness.
Low Severity

Recommendation 5:

Lack of modularity in functions such as `format_sql_result`, as it contains multiple responsibilities.

Suggested Improvement: Refactor functions to adhere to the Single Responsibility Principle, separating formatting from querying logic.
Low Severity

Recommendation 6:

Implicit assumptions in code logic, such as splitting SQL queries to determine table names, can lead to incorrect behavior with complex queries.

Suggested Improvement: Use regex or a SQL parsing library to more reliably extract table names and components from SQL queries.
